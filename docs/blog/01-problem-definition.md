# [1편] 30개 사업장 4대보험 신고, 매달 반복되는 지옥을 자동화하기로 했다

> AI와 노무사가 만드는 4대보험 자동화 시리즈 (1/9)

## 매달 찾아오는 악몽

노무사 사무실에서 30개 이상의 사업장을 관리하고 있다. 프랜차이즈 식당, 소규모 사업장들. 매달 똑같은 업무가 반복된다.

1. 각 사업장에서 급여대장 받기
2. 이번 달 입사자/퇴사자 확인
3. 근로복지공단 사이트 접속
4. 취득신고(입사) / 상실신고(퇴사) 작성
5. 30개 사업장 × 4대보험 = 머리 터짐

특히 월초에는 전화가 빗발친다.

> "이번에 새로 들어온 알바 신고 부탁드려요~"
> "어제 퇴사한 직원 처리해주세요"

## 기존 방식의 한계

근로복지공단에도 **엑셀 일괄신고** 기능이 있다. 하지만:

```
급여대장 (사업장별 양식 다 다름)
    ↓ [수작업 복붙]
일괄신고 엑셀 템플릿 (근로복지공단 양식)
    ↓ [업로드]
근로복지공단 사이트
```

이 **"수작업 복붙"** 과정이 문제다.

- A사업장 급여대장: 이름이 B열, 주민번호가 D열
- B사업장 급여대장: 이름이 C열, 주민번호가 F열
- C사업장 급여대장: 시트 이름도 다르고 시작 행도 다름

30개 사업장이 30개의 다른 양식을 쓴다. 매번 "어디가 이름이더라..." 하면서 찾아야 한다.

## 진짜 원하는 것

```
급여대장 업로드
    ↓ [자동 변환]
신고용 엑셀 다운로드
    ↓ [업로드]
근로복지공단
```

더 나아가면:

```
급여대장 업로드
    ↓ [원클릭]
신고 완료
```

## 왜 직접 만들기로 했나

시중 노무 프로그램들:
- 월 구독료 비쌈 (사업장당 과금)
- 우리 워크플로우에 안 맞음
- 기존 급여대장 양식을 바꿔야 함

**"내 업무 방식에 맞는 도구를 직접 만들자"**

마침 Claude Code라는 AI 코딩 도구를 알게 됐다. 대화하면서 코드를 짜준다고? 한번 해보자.

## 목표 설정

### MVP (최소 기능 제품)

1. **사업장 관리**
   - 사업장 정보 등록 (사업자번호, 관리번호 등)
   - 30개 사업장 관리

2. **급여대장 Import**
   - 엑셀 파일 업로드
   - 사업장별 컬럼 매핑 설정 (한 번만 하면 저장)
   - 근로자 정보 자동 추출

3. **신고서 생성**
   - 해당 월 입사자 → 취득신고 엑셀
   - 해당 월 퇴사자 → 상실신고 엑셀
   - 근로복지공단 양식에 맞게 자동 변환

### 향후 확장

- Chrome 확장프로그램으로 근로복지공단 사이트 자동화
- 클라우드 동기화 (여러 PC에서 사용)
