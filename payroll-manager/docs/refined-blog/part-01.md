# 30개 사업장 엑셀 노가다 탈출기: 노무사가 Claude Code로 4대보험 자동화 시스템을 만들다

> **TL;DR**: 매달 30개 사업장의 4대보험 신고를 수작업으로 처리하던 노무사가, AI 코딩 도구 Claude Code를 활용해 급여대장 자동 변환 시스템을 직접 개발한 이야기입니다. 코딩 경험 없이도 대화만으로 프로그램을 만들 수 있는 시대가 열렸습니다.

---

## 들어가며: 매달 찾아오는 악몽

노무사 사무실에서 30개 이상의 사업장을 관리하고 있습니다. 프랜차이즈 식당, 소규모 제조업체, 서비스업... 매달 똑같은 업무가 반복돼요.

1. 각 사업장에서 급여대장 받기
2. 이번 달 입사자/퇴사자 확인하기
3. 근로복지공단 사이트 접속하기
4. 취득신고(입사) / 상실신고(퇴사) 작성하기
5. 30개 사업장 × 4대보험 = **머리 터짐**

특히 월초에는 전화가 빗발칩니다.

> "이번에 새로 들어온 알바 신고 부탁드려요~"
> "어제 퇴사한 직원 처리해주세요!"

한 달에 수십 건의 입퇴사 처리. 단순 반복 작업이지만, 실수하면 과태료가 부과되는 민감한 업무죠.

---

## 기존 방식의 한계: '복붙 지옥'

근로복지공단에도 **엑셀 일괄신고** 기능이 있습니다. 하지만 문제는 여기서 시작돼요.

```
급여대장 (사업장별 양식 다 다름)
    ↓ [수작업 복붙]
일괄신고 엑셀 템플릿 (근로복지공단 양식)
    ↓ [업로드]
근로복지공단 사이트
```

**"수작업 복붙"** 과정이 바로 병목이에요.

- **A사업장** 급여대장: 이름이 B열, 주민번호가 D열
- **B사업장** 급여대장: 이름이 C열, 주민번호가 F열
- **C사업장** 급여대장: 시트 이름도 다르고 시작 행도 다름

30개 사업장이 30개의 다른 양식을 씁니다. 매번 "어디가 이름이더라..." 하면서 찾아야 하죠. 눈이 빠질 것 같아요.

**[캡처 필요 #1]**: 실제 급여대장 샘플 2~3개 (개인정보 마스킹) - 사업장마다 양식이 다르다는 것을 시각적으로 보여주기

## 진짜 원하는 것: 원클릭 자동화

제가 꿈꾸던 워크플로우는 이랬어요.

```
[현재]
급여대장 업로드 → [수작업 복붙] → 신고용 엑셀 → 근로복지공단

[목표]
급여대장 업로드 → [자동 변환] → 신고용 엑셀 다운로드 → 근로복지공단

[궁극적 목표]
급여대장 업로드 → [원클릭] → 신고 완료!
```

시중에 노무 프로그램들이 있습니다. 하지만:

- **비용 문제**: 월 구독료가 비쌈 (사업장당 과금 방식)
- **호환성 문제**: 우리 워크플로우에 안 맞음
- **전환 비용**: 기존 급여대장 양식을 바꿔야 함

결국 결론은 하나였어요.

> **"내 업무 방식에 맞는 도구를 직접 만들자"**

## Claude Code와의 만남: AI가 코드를 짜준다고?

마침 **Claude Code**라는 AI 코딩 도구를 알게 됐습니다. 대화하면서 코드를 짜준다고? 반신반의하며 시작했어요.

### Claude Code란?

Anthropic에서 만든 AI 코딩 어시스턴트입니다. 터미널에서 대화하듯 요청하면, AI가 코드를 작성하고 실행까지 해줍니다.

```bash
# 설치 후 실행
claude

# 대화 시작
> "급여대장 엑셀 파일을 읽어서 근로복지공단 양식으로 변환하는
   프로그램을 만들어줘"
```

정말 이렇게 간단할까요? 놀랍게도, **실제로 작동합니다**.

물론 한 번에 완벽한 프로그램이 나오진 않아요. 하지만 대화를 통해 점점 구체화하면서, 제가 원하는 기능을 하나씩 추가할 수 있었습니다.

**[캡처 필요 #2]**: Claude Code 터미널 화면 - AI와 대화하며 코드 생성하는 모습

## MVP 목표 설정: 작게 시작하기

처음부터 완벽한 시스템을 만들려고 하면 실패합니다. 그래서 **MVP(최소 기능 제품)**를 먼저 정의했어요.

### 1단계: 사업장 관리
- 사업장 정보 등록 (사업자번호, 관리번호 등)
- 30개 사업장 한눈에 보기

### 2단계: 급여대장 Import
- 엑셀 파일 업로드
- **사업장별 컬럼 매핑 설정** (한 번만 하면 저장)
- 근로자 정보 자동 추출

### 3단계: 신고서 생성
- 해당 월 입사자 → 취득신고 엑셀
- 해당 월 퇴사자 → 상실신고 엑셀
- 근로복지공단 양식에 맞게 자동 변환

### 향후 확장 계획
- Chrome 확장프로그램으로 근로복지공단 사이트 자동화
- 클라우드 동기화 (여러 PC에서 사용)

## 기술 스택 선택: 왜 웹 기반인가?

Claude Code와 상의한 끝에, 다음과 같은 기술 스택을 선택했습니다.

| 기술 | 선택 이유 |
|------|----------|
| **Next.js 15** | React 기반, 빠른 개발 가능 |
| **TypeScript** | 타입 안전성으로 버그 방지 |
| **Tailwind CSS** | 빠른 UI 개발 |
| **Zustand** | 간단한 상태 관리 |
| **xlsx (SheetJS)** | 엑셀 파일 읽기/쓰기 |

**왜 웹 기반인가요?**

- **설치 필요 없음**: 브라우저만 있으면 됨
- **크로스 플랫폼**: Windows, Mac 어디서든
- **업데이트 용이**: 배포만 하면 즉시 반영
- **로컬 우선**: 민감한 데이터(주민번호 등)는 브라우저에만 저장

## 첫 번째 대화: 프로젝트 시작

Claude Code와 나눈 첫 대화입니다.

```
나: "노무사 업무를 자동화하는 웹앱을 만들고 싶어.
     급여대장 엑셀을 업로드하면 4대보험 신고용 엑셀로
     변환해주는 기능이 핵심이야."

Claude: "이해했습니다. 먼저 프로젝트 구조를 만들어볼게요.
         Next.js를 사용하고, 엑셀 처리를 위해 xlsx 라이브러리를
         설치하겠습니다..."
```

**10분 후**, 기본 프로젝트 구조가 완성됐습니다.

```
payroll-manager/
├── src/
│   ├── app/           # 페이지
│   ├── components/    # 재사용 컴포넌트
│   ├── store/         # 상태 관리 (Zustand)
│   └── types/         # TypeScript 타입
├── package.json
└── tailwind.config.js
```

코딩 경험이 없어도, AI와 대화하며 하나씩 만들어갈 수 있었어요.
