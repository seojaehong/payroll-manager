# 사업장 중심 UI: 기능에서 데이터로

> "기능 → 사업장"이 아니라 "사업장 → 기능"으로 사고방식을 바꾸다.

## 기존 구조의 문제

```
기존: 기능 중심
├── /workers (전체 근로자)
├── /wages (전체 급여)
├── /reports (전체 신고서)
└── 사업장 선택은 각 화면에서
```

- 15개 사업장 관리 시 혼란
- 매번 사업장 필터링 필요
- 전체 현황 파악 어려움

## 새로운 구조: 사업장 중심

```
신규: 사업장 중심
├── / (대시보드: 모든 사업장 카드)
└── /businesses/[id] (사업장 상세)
    ├── 근로자 탭
    ├── 급여이력 탭
    ├── 신고서 탭
    └── Import 탭
```

## 대시보드 설계

각 사업장 카드에 핵심 정보 표시:

```tsx
<div className="glass p-6">
  <h3>{business.name}</h3>
  <p>{business.bizNo}</p>

  {/* 재직 현황 */}
  <p>{activeCount}명 재직 중</p>

  {/* 이번달 입퇴사 */}
  <div>입사 {thisMonthJoins} / 퇴사 {thisMonthLeaves}</div>

  {/* 급여 입력률 */}
  <ProgressBar value={wageProgress} />

  {/* 신고 필요 알림 */}
  {needsAcquireReport && <Badge>취득신고 필요</Badge>}
  {needsLoseReport && <Badge>상실신고 필요</Badge>}
</div>
```

## 탭 기반 상세 페이지

하나의 사업장 안에서 모든 작업 완료:

```tsx
const tabs = ['근로자', '급여이력', '신고서', 'Import'];

<Tabs value={activeTab}>
  <WorkersTab />      {/* 근로자 목록, 추가, 수정 */}
  <WagesTab />        {/* 월별 급여 입력 */}
  <ReportsTab />      {/* 신고서 직접 생성 */}
  <ImportTab />       {/* 엑셀 임포트 */}
</Tabs>
```

## 주요 개선점

1. **한 화면에서 완결**: 사업장 선택 후 이탈 없이 모든 작업
2. **현황 파악 용이**: 대시보드에서 15개 사업장 한눈에
3. **신고 필요 알림**: 입퇴사 발생 시 즉시 확인
4. **급여 입력률**: 전년도 데이터 완성도 추적

## 결과

- **작업 동선 단축**: 클릭 수 50% 감소
- **실수 방지**: 사업장 혼동 없음
- **생산성 향상**: 15개 사업장 순차 처리 가능
